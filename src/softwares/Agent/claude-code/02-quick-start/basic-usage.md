# 基础使用

## 首次启动

在项目目录下打开终端，输入 `claude`。首次进入某目录时会询问是否信任，输入 `y` 确认。

## 第一个任务

用 `/plan` 让 AI 先出方案再执行：

```text
/plan 生成一份 Python 脚本，模拟 CaCO3 煅烧过程的 TGA 热重数据，并绘制 TG 和 DTG 曲线图。
```

AI 会先列出方案（任务分解、实现路径），确认后再动手。如果方案有偏差，可以直接告诉它修改，或按 `Ctrl + G` 手动编辑计划。

> 报 `401 Unauthorized` 说明 API Key 未正确配置，检查 `~/.claude/settings.json` 中的 `ANTHROPIC_AUTH_TOKEN`。

## 让 AI 了解项目

用 `/init` 让 Claude 扫描项目目录，自动生成 `CLAUDE.md`。之后每次启动，AI 都能快速了解项目结构和约定。

## 写一个好的 Prompt

模糊的指令可能有效，但通常要花更多时间引导。给 AI 的信息越具体，输出越接近期望。`@文件路径` 可将文件内容注入对话。

**坏例子**：
> 帮我分析一下数据

**好例子**：
> 读取 @data/TGA_raw.csv。画 TG(%) 和 DTG 随温度 T(K) 的变化曲线。TG 用左轴，DTG 用右轴。配色 viridis，线宽 2。保存到 figures/TGA_DTG.pdf。

### 提供验证方法

让 AI 能自行验证结果，而不是等你纠正。

| 而非 | 更好 |
|------|------|
| 帮我画 TGA 对比图 | 画 TG/DTG 双 Y 轴图，核查 α 范围在 [0, 1]，T_onset < T_peak < T_end，不符合则修正 |
| 让仪表板更好看 | 参照截图实现设计，完成后截取效果图对比，列出差异并修正 |
| 构建失败 | 项目构建报错：[粘贴报错]。定位根源并修复，确保构建成功，禁止仅屏蔽报错 |

### 提供具体上下文

引用特定文件、约束和示例，而非让 AI 猜测。

| 而非 | 更好 |
|------|------|
| 为 foo.py 添加测试 | 为 foo.py 编写测试，覆盖用户已注销的边界情况，避免 mock |
| 修复登录错误 | 用户报告会话超时后登录失败，检查 src/auth/ 下的令牌刷新逻辑，写复现测试 |
| 处理这批数据 | 读取 @data/exp-2024/ 下所有 CSV，参考 @scripts/baseline.py 的处理流程，输出相同格式 |

### 直接给材料

用 `@` 引用文件、粘贴截图、贴文档链接，减少 AI 猜测。

```text
@src/auth/login.ts 把这个文件里的 token 刷新逻辑改为静默刷新
```

截图可直接粘贴到对话中。贴官方文档链接比凭印象描述 API 行为有效得多。

## 迭代式工作流

不要期待一次 Prompt 出完美结果：

1. 第一轮给粗需求，看 AI 理解的方向对不对
2. 第二轮纠正偏差，细化需求
3. 第三轮调试优化
