# 使用示例

以下场景示范 Claude Code 各项功能在科研中的使用方式。Prompt 是起点而非模板，根据实际数据和研究目标调整后使用。

## /plan — 先规划再执行

适用于步骤多、易走偏的复杂任务。

```text
/plan 处理 data/raw/ 下的 TG 数据：读取所有 CSV，自动识别温度列和重量列，
归一化为转化率 α，计算 DTG，提取各升温速率下的特征温度（T_onset, T_peak,
T_end），画 TG/DTG 对比图。先给出完整方案，确认后再执行。
```

AI 会先列出文件、观察数据结构、输出方案，确认后逐步执行，每步输出中间结果供检查。

## /goal — 条件驱动的自主循环

适用于有明确验证标准的闭环任务。

```text
/goal 重构 scripts/kinetics_calc.py：修复分峰拟合在高升温速率下发散的问题，
添加边界约束，确保 3 个测试用例全部通过。每轮修改后运行 pytest 验证，
直到所有条件满足。
```

AI 会读取脚本和测试、诊断问题、修改、运行测试、根据结果调整，直到全部通过。

## 从原始数据到可投稿图表

一个完整的 TG 数据处理闭环：

**第 1 步 — 探索数据**
```text
/plan 分析 data/raw/tg-ftir/ 下的 TG-FTIR 联用数据。先看数据结构，
给出分析方案：TG 提取什么参数，FTIR 鉴定哪些官能团，画什么图。
```

**第 2 步 — 处理 TG**
```text
按方案处理 TG 数据：归一化、DTG、提取特征温度，
结果写入 data/processed/tg_summary.csv。
```

**第 3 步 — 画图**
```text
画 FTIR 3D 瀑布图，在特征温度处做切片曲线。
TG 和 DTG 双 Y 轴图，不同升温速率对比。
配色 viridis，字体 Times New Roman 11pt，保存到 figures/。
```

**第 4 步 — 动力学计算**
```text
用 tg_summary.csv 的峰值温度计算 Kissinger 和 FWO 活化能，
画拟合图，标注 R²。
```

**第 5 步 — 提交**
```text
帮我提交所有更改，每个步骤分开提交。
```

## Subagent — 并行处理多组数据

```text
data/raw/ 下有 4 个样品（不同 Ca/S 摩尔比）的 TG 数据，每组独立处理：
归一化、计算 DTG、提取特征温度。4 组相互独立，并行处理，最后汇总比较表。
```

## Git — 小步提交与回滚

```text
帮我提交更改。数据清洗和绘图分开两次提交，用 Conventional Commits 格式。
```

```text
TG 数据处理的结果不对，回滚到上一次提交时的状态。
```

以上示例的共同逻辑：给 AI 明确的目标、输入数据和验收标准，通过"探索 → 执行 → 验证"闭环交付。每步完成后的审查不是多余动作，而是安全使用 Agent 的前提。
